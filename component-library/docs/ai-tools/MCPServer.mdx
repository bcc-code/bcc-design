# MCP Server

MCP server generation is not part of this docs pipeline.

The current AI docs support is intentionally static:

- `/llms.txt`
- `/llms-full.txt`
- `/docs-index.json`
- `/docs/<page>.md`

Use `pnpm run docs:ai` to regenerate those files from the public Storybook docs manifest. Adding an MCP server would require a separate runtime service, authentication model, deployment path, and maintenance plan.
